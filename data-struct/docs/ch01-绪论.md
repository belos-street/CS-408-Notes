## 1.1 数据结构的基本概念

### 1.1.1 基本概念和术语（高频考点）

以下术语为考研高频考点，需精准记忆，避免混淆：

**数据** 是信息的载体，是描述客观事物属性的数、字符及所有能输入到计算机中并被计算机程序识别和处理的符号的集合。

**数据元素** 是数据的基本单位，通常作为一个整体进行考虑和处理。一个数据元素可由若干**数据项**组成，数据项是构成数据元素的不可分割的最小单位。

**数据对象** 是具有相同性质的数据元素的集合，是数据的一个子集。

**数据类型** 是一个值的集合和定义在此集合上的一组操作的总称。按值的不同，数据类型可分为：

- 原子类型：其值不可再分（如整型、字符型）
- 结构类型：其值可以再分解为若干成分（如数组、结构体）
- 抽象数据类型（ADT）：定义了一组抽象的数据操作，不考虑存储结构

**抽象数据类型（ADT）的表示形式**：

ADT通常采用以下格式定义：

```
ADT 类型名 {
    数据对象：<数据对象的定义>
    数据关系：<数据关系的定义>
    基本操作：<基本操作的定义>
}
```

**示例**：线性表的ADT定义

```
ADT LinearList {
    数据对象：D = {aᵢ | aᵢ∈ElemSet, i=1,2,...,n, n≥0}
    数据关系：R = {<aᵢ₋₁, aᵢ> | aᵢ₋₁, aᵢ∈D, i=2,...,n}
    基本操作：
        InitList(&L)        // 初始化线性表
        DestroyList(&L)     // 销毁线性表
        ListInsert(&L, i, e) // 在位置i插入元素e
        ListDelete(&L, i, &e) // 删除位置i的元素
        LocateElem(L, e)    // 查找元素e的位置
        GetElem(L, i, &e)   // 获取位置i的元素
}
```

**数据结构** 是相互之间存在一种或多种特定关系的数据元素的集合。数据结构包括三个方面的内容：逻辑结构、存储结构和数据的运算。

### 1.1.2 数据结构三要素（必考考点）

数据结构的核心三要素：**逻辑结构、物理结构、数据运算**，三者相互关联，是判断数据结构类型的关键。

#### 1. 数据的逻辑结构

逻辑结构是指数据元素之间的逻辑关系，即从逻辑关系上描述数据。逻辑结构分为线性结构和非线性结构：

| 结构类型 | 说明 | 例子 |
|---------|------|------|
| 集合结构 | 元素之间仅存在"同属一个集合"的关系 | 班级学生集合 |
| 线性结构 | 元素之间存在一对一关系 | 线性表、栈、队列、串 |
| 树形结构 | 元素之间存在一对多的层次关系 | 树、二叉树 |
| 图状结构 | 元素之间存在多对多的关系 | 图 |

#### 2. 数据的存储结构（物理结构）

存储结构是指数据结构在计算机中的表示（又称映像），也称物理结构。存储结构主要有以下四种：

**顺序存储**：
- 把逻辑上相邻的元素存储在物理位置相邻的存储单元中
- 优点：可以实现随机存取，每个元素占用最少空间
- 缺点：可能产生**内部碎片**（分配单元内的未用空间），存储空间利用率低；需要预先分配连续空间，不支持动态扩展

**链式存储**：
- 不要求逻辑上相邻的元素在物理位置上也相邻，通过指针域表示元素间的逻辑关系
- 优点：无碎片问题，存储空间利用率高
- 缺点：只能顺序访问，指向后继结点的指针本身也占用存储空间

**索引存储**：
- 在存储元素信息的同时，还建立附加的索引表
- 优点：检索速度快
- 缺点：增加索引表占用额外存储空间，增加和删除数据时需要修改索引表

**散列存储**（哈希存储）：
- 根据元素的关键字直接计算出该元素的存储地址
- 优点：检索、增加和删除操作都很快
- 缺点：散列函数设计不当会导致冲突，增加查询开销

#### 3. 数据的运算

数据的运算是指施加在数据上的操作。常见的数据运算包括：插入、删除、修改、查找、排序等。数据的运算需在逻辑结构上定义，在存储结构上实现。

### 1.1.3 易错点辨析（考研高频陷阱）

- **易错1**：逻辑结构与物理结构混淆——逻辑结构描述"关系"，物理结构描述"存储方式"，例如：链表的逻辑结构是线性结构，物理结构是链式存储。

- **易错2**：数据元素与数据项混淆——数据元素是基本单位，数据项是最小单元（如"学生"是数据元素，"学号"是数据项）。

- **易错3**：抽象数据类型与数据结构混淆——抽象数据类型是"模型+操作"的抽象描述，数据结构是具体的实现（如"线性表"是抽象数据类型，"顺序表、链表"是具体的数据结构）。

- **易错4**：顺序存储产生的是**内部碎片**而非外部碎片——外部碎片是指内存中无法利用的小块空间，链式存储才涉及；顺序存储分配固定大小空间，未用部分为内部碎片。

### 1.1.4 本节试题精选

**选择题**

## 1.2 算法和算法评价

### 1.2.1 算法的基本概念（高频考点）

**算法** 是对特定问题求解步骤的一种描述，它是指令的有限序列。算法必须具备以下五个特性（必考，精准记忆）：

| 特性 | 说明 |
|------|------|
| 有穷性 | 算法必须在执行有限步后结束，每一步都必须在有穷时间内完成 |
| 确定性 | 算法中每一条指令必须有确切的含义，不存在二义性 |
| 可行性 | 算法中描述的操作都可以通过已经实现的基本运算执行有限次来实现 |
| 输入 | 一个算法有零个或多个输入 |
| 输出 | 一个算法有一个或多个输出 |

**算法与程序的区别（易错点）**：算法是"逻辑层面"的步骤集合，程序是"物理层面"的实现（用具体编程语言编写）；算法必须满足有穷性，程序可无限循环（如操作系统，是程序但不是算法）。

**算法的设计要求**：
- 正确性：算法应能够正确地求解问题
- 可读性：算法应具有良好的可读性，以便于理解和修改
- 健壮性：当输入非法数据时，算法也能做出适当反应或处理
- 高效性：算法应有效利用资源，包括时间和空间

### 1.2.2 算法效率的度量（必考考点，重中之重）

算法效率分为**时间效率**（执行时间长短）和**空间效率**（占用存储空间大小），考研核心考查时间复杂度，空间复杂度考查较少但需掌握基础。

#### 1. 时间复杂度（每年必考1题）

一个语句的频度是指该语句在算法中被执行的次数。算法中所有语句的频度之和记为 T(n)，它是该算法问题规模 n 的函数。

**时间复杂度** 用于分析 T(n) 的数量级。常用大O记法表示。

**核心原则**：只关注"增长最快的项"，忽略常数项、低次项和系数（因为当n足够大时，这些项对执行次数的影响可忽略）。

**渐进符号体系**：

| 符号 | 名称 | 含义 | 数学定义 |
|------|------|------|----------|
| O | 大O（上界） | f(n)的增长率**不超过**g(n) | 存在c>0, n₀>0，使∀n≥n₀，f(n)≤c·g(n) |
| Ω | 大Omega（下界） | f(n)的增长率**不低于**g(n) | 存在c>0, n₀>0，使∀n≥n₀，f(n)≥c·g(n) |
| Θ | 大Theta（紧界） | f(n)的增长率**等于**g(n) | f(n) = O(g(n)) 且 f(n) = Ω(g(n)) |
| o | 小o | f(n)的增长率**严格小于**g(n) | 对于任意c>0，存在n₀>0，使∀n≥n₀，f(n)<c·g(n) |
| ω | 小omega | f(n)的增长率**严格大于**g(n) | 对于任意c>0，存在n₀>0，使∀n≥n₀，f(n)>c·g(n) |

**注意**：408考试中主要考查大O表示法，但需理解O、Ω、Θ的关系。

**分析方法**：
1. 找出所有语句中频度最大的语句（通常是嵌套最深的循环体）
2. 计算该语句的频度，得到 T(n)
3. 用大O记法表示 T(n)

**常见时间复杂度**：
$$O(1) < O(\log_2 n) < O(n) < O(n\log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$$

**最好、最坏和平均情况**：
- 最好情况：算法在最好情况下的时间复杂度
- 最坏情况：算法在最差情况下的时间复杂度（考研中时间复杂度均按"最坏情况"计算）
- 平均情况：所有可能输入在等概率情况下算法的时间复杂度

**递归算法时间复杂度分析方法**：

递归算法的时间复杂度通常通过建立递推公式来求解。

**方法一：递推公式法**
- 写出递推关系式 T(n) = aT(n/b) + f(n)
- 求解递推式得到时间复杂度

**示例1**：计算斐波那契数列的递归算法
```c
int Fib(int n) {
    if (n <= 1) return n;
    return Fib(n-1) + Fib(n-2);
}
```
递推公式：T(n) = T(n-1) + T(n-2) + O(1)
解得：T(n) = O(2ⁿ)（指数级，效率很低）

**示例2**：二分查找的递归算法
```c
int BinarySearch(int A[], int low, int high, int key) {
    if (low > high) return -1;
    int mid = (low + high) / 2;
    if (A[mid] == key) return mid;
    else if (A[mid] > key) return BinarySearch(A, low, mid-1, key);
    else return BinarySearch(A, mid+1, high, key);
}
```
递推公式：T(n) = T(n/2) + O(1)
解得：T(n) = O(log₂n)

**方法二：主定理（Master Theorem）**

对于递推式 T(n) = aT(n/b) + f(n)，其中 a≥1, b>1：

| 情况 | 条件 | 时间复杂度 |
|------|------|-----------|
| 情况1 | f(n) = O(n^(log_b a - ε))，ε>0 | T(n) = Θ(n^(log_b a)) |
| 情况2 | f(n) = Θ(n^(log_b a) · (log n)^k)，k≥0 | T(n) = Θ(n^(log_b a) · (log n)^(k+1)) |
| 情况3 | f(n) = Ω(n^(log_b a + ε))，ε>0 | T(n) = Θ(f(n)) |

**注意**：408考试中递归复杂度分析主要考查简单递推，主定理了解即可。

#### 2. 空间复杂度

**空间复杂度** S(n) 是算法执行过程中**所需存储空间的大小**（与输入规模 n 的函数关系），记为 S(n) = O(g(n))。

**核心考查**：额外存储空间（不包括输入数据本身占用的空间），分为2类：

- **固定空间**：与n无关，如变量、常量占用的空间，复杂度O(1)
- **可变空间**：与n相关，如数组、递归栈占用的空间（如递归实现的斐波那契数列，递归深度为n，空间复杂度O(n)）

**原地工作**：若算法执行时所需的辅助空间相对于输入数据量是常数，则称该算法为原地工作，辅助空间复杂度为 O(1)。

### 1.2.3 易错点辨析（考研高频陷阱）

- **易错1**：时间复杂度计算忽略"最坏情况"——考研中时间复杂度均按"最坏情况"计算（如顺序查找，最坏情况是查找最后一个元素，复杂度O(n)）。

- **易错2**：混淆"基本操作"——基本操作是执行次数最多的操作，而非任意操作（如嵌套循环中，内层循环的操作是基本操作）。

- **易错3**：认为"循环次数多，复杂度就高"——复杂度取决于与n的关系，如O(n)的循环（n=1000）执行次数比O(1)的操作（执行100次）多，但复杂度更低。

- **易错4**：空间复杂度包含输入数据的空间——考研中空间复杂度仅计算"额外空间"，输入数据本身的空间不计入。

- **易错5**：递归算法空间复杂度都是O(1)——递归算法空间复杂度通常为**O(递归深度)**，递归调用需要栈空间保存现场。

### 1.2.4 本节试题精选

**选择题**

1. 下面关于算法说法正确的是（ ）
   - A. 算法原地工作的含义是程序不需要额外的空间
   - B. 算法的有穷性是指算法程序的运行时间是有限的
   - C. 算法的空间复杂度是指执行这个算法所需的内存空间
   - D. 以上都不对

2. 设n为正整数，分析以下代码的时间复杂度：
   ```c
   for(i = 2; i <= n; i++) {
       for(j = 2; j <= i; j++) {
           x = x + 1;
       }
   }
   ```
   - A. O(n)
   - B. O(n²)
   - C. O(n³)
   - D. O(n²logn)

3. 以下算法的时间复杂度是（ ）
   ```c
   i = 1;
   while(i <= n) {
       i = i * 2;
   }
   ```
   - A. O(n)
   - B. O(log₂n)
   - C. O(n²)
   - D. O(1)

4. 已知两个长度为n的顺序表，若将它们合并成一个新的顺序表，最坏情况下的时间复杂度为（ ）
   - A. O(n)
   - B. O(n²)
   - C. O(nlogn)
   - D. O(n+m)，其中m为另一个表长

5. 下列程序段的时间复杂度是（ ）
   ```c
   count = 0;
   for(k = 1; k <= n; k *= 2) {
       for(j = 1; j <= n; j++) {
           count++;
       }
   }
   ```
   - A. O(n)
   - B. O(nlog₂n)
   - C. O(n²)
   - D. O(logn)

**答案与解析**

1. **答案：C**
   - 解析：算法原地工作是指算法执行时所需的辅助空间是常数；算法的有穷性是指算法必须在执行有限步后结束，而不是程序运行时间；算法的空间复杂度是指执行这个算法所需的内存空间。

2. **答案：B**
   - 解析：外层循环 i 从 2 到 n，内层循环 j 从 2 到 i。语句 `x = x + 1;` 的频度为 Σ(i-1)，i 从 2 到 n，即 (2-1)+(3-1)+...+(n-1) = (n-1)n/2 = (n²-n)/2，时间复杂度为 O(n²)。

3. **答案：B**
   - 解析：循环条件 i <= n，每次循环 i 乘以 2，即 i = 2⁰, 2¹, 2², ..., 2ᵏ。循环执行 k+1 次后 i > n，即 2ᵏ > n，k > log₂n，所以时间复杂度为 O(log₂n)。

4. **答案：A**
   - 解析：合并两个长度为 n 的顺序表，最坏情况下需要比较所有元素，时间复杂度为 O(n)。

5. **答案：B**
   - 解析：外层循环 k 从 1 开始，每次乘以 2，即 k = 1, 2, 4, 8, ..., 2ᵐ ≤ n，执行 m+1 次，m = ⌊log₂n⌋。内层循环 j 从 1 到 n，执行 n 次。总执行次数为 n·(m+1) = O(nlog₂n)。

**综合应用题**

1. 设n为正整数，求以下程序段的时间复杂度和空间复杂度：
   ```c
   void func(int n) {
       int i, j, k, count = 0;
       for(i = 1; i <= n; i++) {
           for(j = 1; j <= n; j++) {
               for(k = 1; k <= n; k++) {
                   if(i * j * k == n) count++;
               }
           }
       }
   }
   ```
   
   **答案与解析**：
   - 时间复杂度：O(n³)。三重嵌套循环，每层循环执行n次，基本操作`if(i * j * k == n) count++;`执行n³次。
   - 空间复杂度：O(1)。只使用了常数个辅助变量（i, j, k, count）。

2. 某算法的时间复杂度为O(n²)，在n=1000时运行时间为1秒。问：n=2000时，运行时间约为多少？
   
   **答案与解析**：
   - 答案：约4秒。
   - 解析：时间复杂度为O(n²)，运行时间与n²成正比。n从1000变为2000，n²从10⁶变为4×10⁶，是原来的4倍，因此运行时间约为4秒。

3. 分析以下递归算法的时间复杂度和空间复杂度：
   ```c
   int func(int n) {
       if (n <= 1) return 1;
       return n * func(n - 1);
   }
   ```
   
   **答案与解析**：
   - 时间复杂度：O(n)。递归调用n次，每次递归执行常数时间操作。
   - 空间复杂度：O(n)。递归深度为n，需要n个栈帧空间。

4. 给定以下程序段，分析其在最好情况、最坏情况和平均情况下的时间复杂度：
   ```c
   int search(int A[], int n, int key) {
       for(int i = 0; i < n; i++) {
           if(A[i] == key) return i;
       }
       return -1;
   }
   ```
   
   **答案与解析**：
   - 最好情况：O(1)。key在数组第一个位置，只需比较1次。
   - 最坏情况：O(n)。key在数组最后一个位置或不存在，需要比较n次。
   - 平均情况：O(n)。假设key在数组中每个位置的概率相等，平均比较次数为(n+1)/2。

## 归纳总结

本章介绍了数据结构的基本概念和算法评价的基础知识，是整个数据结构课程的奠基性章节。

**核心考点**：
1. **数据结构三要素**：逻辑结构（线性/非线性）、存储结构（顺序/链式/索引/散列）、数据运算
2. **算法特性**：有穷性、确定性、可行性、输入、输出
3. **时间复杂度**：大O记法，最好/最坏/平均情况分析
4. **空间复杂度**：原地工作算法空间复杂度为 O(1)

**典型错误辨析**：

| 错误说法 | 正确说法 | 解析 |
|---------|---------|------|
| 顺序存储产生外部碎片 | 顺序存储产生**内部碎片** | 外部碎片是指内存中无法利用的小块空间，链式存储才涉及；顺序存储分配固定大小空间，未用部分为内部碎片 |
| 时间复杂度只看最外层循环 | 应找**最深嵌套的基本操作** | 时间复杂度由执行次数最多的语句决定，通常是嵌套最深的循环体 |
| O(n²)一定比O(nlogn)慢 | 常数因子和n的规模会影响实际运行时间 | 当n较小时，O(n²)可能更快；且O(nlogn)的常数因子可能较大 |
| 递归算法空间复杂度都是O(1) | 递归算法空间复杂度通常为**O(递归深度)** | 递归调用需要栈空间保存现场，深度为n的递归空间复杂度为O(n) |
| 算法原地工作不需要任何额外空间 | 原地工作是指**辅助空间为常数** | 原地工作算法可以使用常数个辅助变量，空间复杂度为O(1) |

## 思维拓展

**问题**：若算法A的时间复杂度为 O(n²)，算法B的时间复杂度为 O(nlog₂n)，在什么情况下选择算法A而非算法B？

**思考方向**：
1. 考虑常数因子：O(nlog₂n) 的常数因子可能比 O(n²) 大，在 n 较小时可能更慢
2. 考虑实际问题的规模：若 n 很小，两种算法差别不大
3. 考虑实现复杂度：算法A可能更简单、更易维护
4. 考虑最坏情况与平均情况：某些 O(n²) 算法在平均情况下表现更好
