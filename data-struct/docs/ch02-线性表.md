## 2.1 线性表的定义和基本操作

### 2.1.1 线性表的定义（高频考点）

**线性表** 是具有相同数据类型的 n（n≥0）个数据元素的有限序列。线性表的特点如下：

- 表中元素个数 n 称为线性表的长度
- 当 n=0 时，线性表为空表
- 表中第一个元素称为**表头元素**，最后一个元素称为**表尾元素**
- 除表头元素外，每个元素有且仅有一个直接前驱
- 除表尾元素外，每个元素有且仅有一个直接后继

**线性表的基本概念**：

| 概念 | 说明 |
|------|------|
| 表长 | 线性表中数据元素的个数 |
| 空表 | 长度为0的线性表 |
| 表头元素 | 线性表中的第一个元素 |
| 表尾元素 | 线性表中的最后一个元素 |
| 直接前驱 | 某个元素前面的相邻元素 |
| 直接后继 | 某个元素后面的相邻元素 |

### 2.1.2 线性表的基本操作（重要）

线性表的基本操作包括：

| 操作 | 说明 |
|------|------|
| `InitList(&L)` | 初始化线性表，构造一个空的线性表 L |
| `DestroyList(&L)` | 销毁线性表，释放线性表 L 占用的内存空间 |
| `ListInsert(&L, i, e)` | 在线性表 L 的第 i 个位置插入元素 e |
| `ListDelete(&L, i, &e)` | 删除线性表 L 的第 i 个元素，并用 e 返回其值 |
| `LocateElem(L, e)` | 在线性表 L 中查找第一个值为 e 的元素，返回其位序 |
| `GetElem(L, i, &e)` | 获取线性表 L 中第 i 个元素的值，用 e 返回 |
| `Length(L)` | 求线性表 L 的长度 |
| `Empty(L)` | 判断线性表 L 是否为空表 |
| `ClearList(&L)` | 清空线性表 L（保留表结构） |

### 2.1.3 本节试题精选

**选择题**

## 2.2 线性表的顺序表示

### 2.2.1 顺序表的定义（高频考点）

**顺序表** 是用一组地址连续的存储单元依次存储线性表中的数据元素，从而使得逻辑上相邻的元素在物理位置上也相邻。

**顺序表的特点**：
- 顺序表中元素的逻辑顺序与物理顺序一致
- 可以通过首地址和元素序号（下标）直接访问任意元素，即**随机访问**
- 存储密度高（只存储数据元素本身）
- 插入和删除操作需要移动大量元素

**顺序表的静态分配**（使用数组）：

```c
#define MAXSIZE 100  // 定义顺序表的最大长度

typedef struct {
    ElemType data[MAXSIZE];  // 存储数据元素的数组
    int length;              // 当前顺序表的长度
} SqList;                    // 顺序表的类型定义
```

**顺序表的动态分配**（使用指针和 malloc）：

```c
typedef struct {
    ElemType *elem;  // 指向数据元素的指针
    int length;      // 当前顺序表的长度
    int listsize;    // 当前分配的存储容量
} SqList;
```

动态分配顺序表初始化时需使用 `malloc` 分配存储空间：

```c
L.elem = (ElemType *)malloc(sizeof(ElemType) * INITSIZE);
if (L.elem == NULL) {
    // 内存分配失败处理
}
L.length = 0;
L.listsize = INITSIZE;
```

### 2.2.2 顺序表上基本操作的实现（必考考点，重点掌握）

#### 1. 顺序表的插入操作

```c
// 在顺序表 L 的第 i 个位置插入元素 e
// i 的取值范围为 1 ≤ i ≤ L.length + 1
bool ListInsert(SqList &L, int i, ElemType e) {
    if (i < 1 || i > L.length + 1)  // 检查插入位置的合法性
        return false;
    if (L.length >= MAXSIZE)  // 检查顺序表是否已满
        return false;
    for (int j = L.length; j >= i; j--)  // 将第 i 个位置之后的元素后移
        L.data[j] = L.data[j - 1];
    L.data[i - 1] = e;  // 插入元素
    L.length++;  // 表长加 1
    return true;
}
```

**时间复杂度分析**：
- 最好情况：在表尾插入（i = L.length + 1），无需移动元素，时间复杂度 O(1)
- 最坏情况：在表头插入（i = 1），需移动所有 n 个元素，时间复杂度 O(n)
- 平均情况：元素移动次数约为 n/2，时间复杂度 O(n)

#### 2. 顺序表的删除操作

```c
// 删除顺序表 L 的第 i 个元素，并用 e 返回其值
bool ListDelete(SqList &L, int i, ElemType &e) {
    if (i < 1 || i > L.length)  // 检查删除位置的合法性
        return false;
    e = L.data[i - 1];  // 保存被删除元素的值
    for (int j = i; j < L.length; j++)  // 将第 i 个位置之后的元素前移
        L.data[j - 1] = L.data[j];
    L.length--;  // 表长减 1
    return true;
}
```

**时间复杂度分析**：
- 最好情况：删除表尾元素（i = L.length），无需移动元素，时间复杂度 O(1)
- 最坏情况：删除表头元素（i = 1），需移动 n-1 个元素，时间复杂度 O(n)
- 平均情况：元素移动次数约为 (n-1)/2，时间复杂度 O(n)

#### 3. 顺序表的按值查找

```c
// 在顺序表 L 中查找第一个值为 e 的元素，返回其位序
int LocateElem(SqList L, ElemType e) {
    for (int i = 0; i < L.length; i++) {
        if (L.data[i] == e)
            return i + 1;  // 返回位序（从 1 开始）
    }
    return 0;  // 查找失败
}
```

**时间复杂度分析**：
- 最好情况：查找的元素在表头，时间复杂度 O(1)
- 最坏情况：查找的元素在表尾或不存在，时间复杂度 O(n)
- 平均情况：比较次数约为 n/2，时间复杂度 O(n)

#### 4. 顺序表的初始化

```c
// 初始化顺序表
void InitList(SqList &L) {
    L.data = (ElemType *)malloc(sizeof(ElemType) * MAXSIZE);
    if (L.data == NULL) {
        // 内存分配失败处理
        exit(1);
    }
    L.length = 0;
    L.listsize = MAXSIZE;
}
```

#### 5. 顺序表求表长

```c
// 返回顺序表 L 的长度
int Length(SqList L) {
    return L.length;
}
```

#### 6. 判断顺序表是否为空

```c
// 判断顺序表 L 是否为空，若为空返回 true，否则返回 false
bool Empty(SqList L) {
    return L.length == 0;
}
```

#### 7. 顺序表的销毁

```c
// 销毁顺序表 L，释放内存空间
void DestroyList(SqList &L) {
    if (L.data != NULL) {
        free(L.data);  // 释放动态分配的内存
        L.data = NULL;
    }
    L.length = 0;
    L.listsize = 0;
}
```

#### 8. 顺序表的清空

```c
// 清空顺序表 L（保留结构，仅重置长度）
void ClearList(SqList &L) {
    L.length = 0;  // 只需将长度置为 0
}
```

### 2.2.3 易错点辨析（考研高频陷阱）

- **易错1**：顺序表插入位置合法性判断——第 i 个位置对应下标 i-1，插入位置 i 的合法范围是 1 到 L.length+1
- **易错2**：顺序表插入/删除时元素移动方向——插入时从后往前移动，删除时从前向后移动
- **易错3**：顺序表支持随机访问——通过下标可在 O(1) 时间内访问任意元素，这是顺序表相对于链表的主要优势
- **易错4**：动态分配顺序表的内存释放——使用完毕后需调用 `free(L.elem)` 释放内存

### 2.2.4 本节试题精选

**选择题**

1. 以下哪个是顺序表的优点？（ ）
   - A. 插入删除操作方便
   - B. 存储密度高，可随机访问
   - C. 不需要存储指针域
   - D. B 和 C

2. 在长度为 n 的顺序表中删除第 i 个元素（1 ≤ i ≤ n），需要移动（ ）个元素。
   - A. i
   - B. n-i
   - C. n-i+1
   - D. n-i-1

3. 在长度为 n 的顺序表的第 i 个位置插入一个元素，需要移动（ ）个元素。
   - A. i
   - B. n-i
   - C. n-i+1
   - D. n-i-1

4. 顺序表所占用的存储空间大小（ ）
   - A. 固定不变
   - B. 与表长成正比
   - C. 与元素类型无关
   - D. 与存储地址相关

5. 以下关于顺序表的说法错误的是（ ）
   - A. 顺序表的插入和删除操作时间复杂度一定为 O(n)
   - B. 顺序表可以随机访问任意元素
   - C. 顺序表的存储密度比链表高
   - D. 顺序表需要预先分配连续存储空间

**答案与解析**

1. **答案：D**
   - 解析：顺序表的主要优点是存储密度高（不需要存储指针域）和可随机访问；插入删除操作不方便，需要移动大量元素。

2. **答案：B**
   - 解析：删除第 i 个元素后，第 i+1 个元素到第 n 个元素需要前移，共移动 n-i 个元素。

3. **答案：C**
   - 解析：在第 i 个位置插入元素，需要将第 i 个位置到第 n 个位置的元素后移，共移动 n-i+1 个元素。

4. **答案：B**
   - 解析：顺序表占用存储空间 = 数据元素占用空间 + 存储空间利用率相关空间，与表长成正比。

5. **答案：A**
   - 解析：顺序表的插入和删除操作，在表尾执行时时间复杂度为 O(1)，只有在表头或中间执行时才是 O(n)。

## 2.3 线性表的链式表示

### 2.3.1 单链表的定义（高频考点）

**单链表** 是线性表的链式存储结构，通过指针将一系列分散的存储单元链接起来。每个结点包含**数据域**和**指针域**，指针域存储后继结点的地址。

**单链表的结点结构**：

```c
typedef struct LNode {
    ElemType data;      // 数据域
    struct LNode *next; // 指针域，指向后继结点
} LNode, *LinkList;
```

**头结点**：为了操作方便，在单链表第一个结点之前附加一个结点，称为头结点。头结点的数据域可以为空或不存储任何信息，指针域存储指向第一个结点的指针。

**头结点的作用**：
- 使单链表的插入和删除操作统一，无需单独处理空表
- 使第一个结点的操作与其它结点一致

**单链表的特点**：
- 不支持随机访问，只能从头指针开始顺序访问
- 存储密度低于顺序表（需存储指针域）
- 插入和删除操作只需修改指针，时间复杂度 O(1)（不计查找位置的时间）

### 2.3.2 单链表上基本操作的实现（必考考点，重点掌握）

#### 1. 单链表的初始化

```c
// 初始化带头结点的单链表
bool InitList(LinkList &L) {
    L = (LNode *)malloc(sizeof(LNode));  // 创建头结点
    if (L == NULL)  // 内存分配失败
        return false;
    L->next = NULL;  // 头结点指针域置空
    return true;
}
```

#### 2. 求单链表的表长

```c
// 返回单链表 L 的长度（不计头结点）
int Length(LinkList L) {
    int len = 0;
    LNode *p = L->next;  // p 指向第一个结点
    while (p != NULL) {
        len++;
        p = p->next;
    }
    return len;
}
```

#### 3. 单链表的按序号查找

```c
// 在单链表 L 中查找第 i 个结点，返回指向该结点的指针
LNode *GetElem(LinkList L, int i) {
    if (i < 1)  // i 值不合法
        return NULL;
    LNode *p = L;  // p 指向头结点
    int j = 0;
    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }
    return p;  // 若 i 大于表长，p 为 NULL
}
```

**时间复杂度**：O(n)

#### 4. 单链表的按值查找

```c
// 在单链表 L 中查找第一个值为 e 的结点，返回指向该结点的指针
LNode *LocateElem(LinkList L, ElemType e) {
    LNode *p = L->next;  // p 指向第一个结点
    while (p != NULL && p->data != e)
        p = p->next;
    return p;  // 查找失败返回 NULL
}
```

**时间复杂度**：O(n)

#### 5. 单链表的插入操作

```c
// 在单链表 L 的第 i 个位置插入元素 e
bool ListInsert(LinkList &L, int i, ElemType e) {
    if (i < 1)
        return false;
    LNode *p = GetElem(L, i - 1);  // 找到第 i-1 个结点
    if (p == NULL)  // i 值不合法
        return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));  // 创建新结点
    s->data = e;
    s->next = p->next;  // 修改指针
    p->next = s;
    return true;
}
```

**时间复杂度**：O(n)（主要耗时在查找第 i-1 个结点）

#### 6. 前插操作（在指定结点 p 前插入）

```c
// 在给定结点 p 前插入元素 e（未给出头指针时使用）
bool InsertPriorNode(LNode *p, ElemType e) {
    if (p == NULL)
        return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));
    s->data = p->data;  // 将 p 的数据复制到 s
    s->next = p->next;
    p->data = e;  // 将 e 复制到 p
    p->next = s;
    return true;
}
```

#### 7. 单链表的删除操作

```c
// 删除单链表 L 的第 i 个结点，并用 e 返回被删元素的值
bool ListDelete(LinkList &L, int i, ElemType &e) {
    if (i < 1)
        return false;
    LNode *p = GetElem(L, i - 1);  // 找到第 i-1 个结点
    if (p == NULL || p->next == NULL)  // i 值不合法
        return false;
    LNode *q = p->next;  // q 指向被删结点
    e = q->data;
    p->next = q->next;  // 修改指针
    free(q);  // 释放被删结点
    return true;
}
```

**时间复杂度**：O(n)

#### 8. 删除指定结点

```c
// 删除给定结点 p（假设 p 不是最后一个结点）
bool DeleteNode(LNode *p) {
    if (p == NULL)
        return false;
    LNode *q = p->next;  // q 指向 p 的后继
    p->data = q->data;  // 用后继结点的值覆盖
    p->next = q->next;  // 修改指针
    free(q);  // 释放后继结点
    return true;
}
```

**注意**：若 p 是最后一个结点，需从表头开始查找其前驱，时间复杂度 O(n)

#### 9. 单链表的遍历

```c
// 遍历单链表 L，输出每个结点的值
void TraverseList(LinkList L) {
    LNode *p = L->next;  // p 指向第一个结点
    while (p != NULL) {
        printf("%d ", p->data);
        p = p->next;
    }
    printf("\n");
}
```

#### 10. 单链表的销毁

```c
// 销毁单链表 L，释放所有结点的内存
void DestroyList(LinkList &L) {
    LNode *p = L;
    while (p != NULL) {
        LNode *q = p;      // q 指向待释放结点
        p = p->next;       // p 移向下一结点
        free(q);           // 释放 q 结点
    }
    L = NULL;  // 头指针置空
}
```

#### 11. 单链表的清空

```c
// 清空单链表 L（保留头结点，释放所有数据结点）
void ClearList(LinkList &L) {
    LNode *p = L->next;  // p 指向第一个数据结点
    while (p != NULL) {
        LNode *q = p;
        p = p->next;
        free(q);
    }
    L->next = NULL;  // 头结点指针域置空
}
```

### 2.3.3 双链表（重要）

双链表的结点包含两个指针域，分别指向直接前驱和直接后继。

**双链表的结点结构**：

```c
typedef struct DNode {
    ElemType data;           // 数据域
    struct DNode *prior;      // 指向前驱结点
    struct DNode *next;      // 指向后继结点
} DNode, *DLinkList;
```

**双链表的插入操作**（在结点 p 后插入 s）：

```c
s->next = p->next;
if (p->next != NULL)  // p 有后继结点
    p->next->prior = s;
s->prior = p;
p->next = s;
```

**双链表的删除操作**（删除结点 p 的后继 q）：

```c
q = p->next;
p->next = q->next;
if (q->next != NULL)  // q 有后继结点
    q->next->prior = p;
free(q);
```

**双链表的特点**：
- 支持双向遍历
- 插入和删除操作更方便（可直接找到前驱）
- 占用更多存储空间（两个指针域）

### 2.3.4 循环链表（重要）

**循环单链表**：将单链表最后一个结点的指针域指向头结点，形成环。

```c
// 初始化循环单链表
bool InitList(LinkList &L) {
    L = (LNode *)malloc(sizeof(LNode));
    if (L == NULL)
        return false;
    L->next = L;  // 指针指向自己，形成环
    return true;
}

// 判断循环单链表是否为空
bool Empty(LinkList L) {
    return L->next == L;
}

// 判断结点 p 是否是表尾结点
bool IsTail(LinkList L, LNode *p) {
    return p->next == L;
}
```

**循环双链表**：将双链表的表头结点的 prior 指向表尾结点，表尾结点的 next 指向头结点。

```c
// 初始化循环双链表
bool InitList(DLinkList &L) {
    L = (DNode *)malloc(sizeof(DNode));
    if (L == NULL)
        return false;
    L->next = L;   // 头结点的 next 指向自己
    L->prior = L;  // 头结点的 prior 指向自己
    return true;
}
```

### 2.3.5 静态链表（了解）

静态链表借助数组实现，用下标代替指针来连接各结点。

```c
#define MAXSIZE 100

typedef struct {
    ElemType data;  // 数据域
    int next;        // 指针域（存放下一个结点在数组中的下标）
} SLinkList[MAXSIZE];
```

静态链表的特点：
- 不需要指针，适合在没有指针的语言中实现链表功能
- 需要预先分配较大的连续空间
- 插入和删除操作只需修改 next 指针，无需移动元素

### 2.3.6 顺序表和链表的比较（必考考点，重点掌握）

| 特性 | 顺序表 | 链表 |
|------|--------|------|
| 存储方式 | 连续存储单元 | 分散存储，通过指针链接 |
| 存储密度 | 高（1） | 低（需存储指针） |
| 随机访问 | 支持，O(1) | 不支持，需遍历，O(n) |
| 插入删除 | 需移动元素，O(n) | 只需修改指针，O(1)（不计查找位置） |
| 空间分配 | 需预先分配连续空间 | 动态分配，灵活 |
| 扩展容量 | 不方便 | 方便 |
| 适用场景 | 频繁查询，较少插入删除 | 频繁插入删除，较少查询 |

**存储密度** = 数据元素本身占用的存储空间 / 结点占用的总空间

**选择原则**：
- 若程序需要频繁地按序号访问元素，选择顺序表
- 若程序需要频繁地插入和删除元素，选择链表
- 若线性表长度变化较大，应选择链表
- 若线性表长度基本稳定，选择顺序表

### 2.3.7 易错点辨析（考研高频陷阱）

- **易错1**：链表不支持随机访问——只能顺序访问，访问第 i 个元素必须从头遍历
- **易错2**：单链表插入/删除的时间复杂度——修改指针本身是 O(1)，但查找第 i-1 个元素是 O(n)
- **易错3**：头结点与头指针混淆——头指针是指向链表的指针，头结点是链表的第一个结点（可带可不带）
- **易错4**：循环链表的判断——循环链表不是空表，它仍然有表长，只是最后一个结点的指针指向头结点
- **易错5**：链表释放内存——插入操作需 `malloc`，删除操作需 `free`

**1. 顺序表的逆置（2010 年 408 真题改编）**

设计一个算法，将顺序表 L 中的所有元素逆置。要求：
- 算法的时间复杂度为 O(n)
- 空间复杂度为 O(1)

**答案与解析**：

```c
// 方法：使用两个指针，从两端向中间移动，交换元素
void Reverse(SqList &L) {
    int i, j;
    ElemType temp;
    for (i = 0, j = L.length - 1; i < j; i++, j--) {
        // 交换 L.data[i] 和 L.data[j]
        temp = L.data[i];
        L.data[i] = L.data[j];
        L.data[j] = temp;
    }
}
```

**时间复杂度**：O(n)，需要遍历 n/2 个元素  
**空间复杂度**：O(1)，只使用了常数个辅助变量

**2. 单链表的逆置（2012 年 408 真题）**

设计一个算法，将带头结点的单链表 L 逆置。要求：
- 在原链表上进行操作，不创建新结点
- 时间复杂度为 O(n)

**答案与解析**：

```c
// 方法：头插法重建链表
void Reverse(LinkList &L) {
    LNode *p, *q;
    p = L->next;  // p 指向第一个数据结点
    L->next = NULL;  // 头结点的指针域置空
    
    while (p != NULL) {
        q = p->next;  // q 保存 p 的后继
        p->next = L->next;  // 将 p 插入到头结点之后
        L->next = p;
        p = q;  // p 移向下一结点
    }
}
```

**时间复杂度**：O(n)，遍历链表一次  
**空间复杂度**：O(1)，只使用了常数个辅助指针

**3. 两个有序链表的合并（2015 年 408 真题改编）**

设有两个带头结点的单链表 L1 和 L2，元素均按递增有序排列。设计一个算法，将 L1 和 L2 合并为一个按递增有序的单链表 L，要求：
- 利用原链表的结点，不创建新结点
- 合并后 L1 和 L2 不再使用

**答案与解析**：

```c
// 方法：归并思想，比较两个链表的当前结点，将较小的结点尾接到结果链表
LinkList Merge(LinkList &L1, LinkList &L2) {
    LNode *p1 = L1->next;  // p1 指向 L1 的第一个结点
    LNode *p2 = L2->next;  // p2 指向 L2 的第一个结点
    LNode *r;  // r 指向结果链表的尾结点
    
    LinkList L = L1;  // 用 L1 的头结点作为结果链表的头结点
    r = L;
    L1->next = NULL;
    
    // 归并两个链表
    while (p1 != NULL && p2 != NULL) {
        if (p1->data <= p2->data) {
            r->next = p1;
            r = p1;
            p1 = p1->next;
        } else {
            r->next = p2;
            r = p2;
            p2 = p2->next;
        }
    }
    
    // 将剩余部分接上
    if (p1 != NULL)
        r->next = p1;
    if (p2 != NULL)
        r->next = p2;
    
    free(L2);  // 释放 L2 的头结点
    return L;
}
```

**时间复杂度**：O(m+n)，m 和 n 分别为两个链表的长度  
**空间复杂度**：O(1)，只使用了常数个辅助指针

**4. 删除有序链表中的重复元素**

设有一个带头结点的单链表 L，元素按递增有序排列。设计一个算法，删除链表中所有值重复的结点（只保留一个）。

**答案与解析**：

```c
// 方法：遍历链表，若当前结点与后继结点值相同，则删除后继结点
void DeleteDuplicates(LinkList &L) {
    LNode *p = L->next;  // p 指向第一个数据结点
    LNode *q;
    
    while (p != NULL && p->next != NULL) {
        if (p->data == p->next->data) {
            q = p->next;  // q 指向待删除结点
            p->next = q->next;  // 修改指针
            free(q);  // 释放结点
        } else {
            p = p->next;  // p 移向下一结点
        }
    }
}
```

**时间复杂度**：O(n)，遍历链表一次  
**空间复杂度**：O(1)

**5. 查找链表的中间结点**

设计一个算法，查找带头结点的单链表 L 的中间结点。若链表长度为偶数，则返回中间两个结点的后一个。要求：
- 只能遍历链表一次
- 空间复杂度为 O(1)

**答案与解析**：

```c
// 方法：快慢指针法，快指针每次走两步，慢指针每次走一步
LNode *FindMiddle(LinkList L) {
    LNode *slow = L->next;  // 慢指针
    LNode *fast = L->next;  // 快指针
    
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;      // 慢指针走一步
        fast = fast->next->next;  // 快指针走两步
    }
    
    return slow;  // slow 指向中间结点
}
```

**时间复杂度**：O(n)，遍历链表一次  
**空间复杂度**：O(1)

**解题思路**：当快指针到达表尾时，慢指针刚好走到链表中间位置。

## 归纳总结

本章介绍了线性表的两种存储结构：顺序存储（顺序表）和链式存储（链表），是数据结构课程的核心基础章节。

**核心考点**：
1. **线性表的定义**：n 个相同数据类型数据元素的有限序列
2. **顺序表**：随机访问 O(1)，插入删除 O(n)，存储密度高
3. **单链表**：顺序访问，插入删除 O(1)（不计查找位置），需存储指针
4. **双链表**：可双向遍历，插入删除更方便
5. **循环链表**：表尾指针指向头结点，可从任意结点访问全表
6. **顺序表 vs 链表**：存储方式、访问方式、操作效率、适用场景的对比

**典型错误辨析**：

| 错误说法 | 正确说法 | 解析 |
|---------|---------|------|
| 顺序表插入删除时间复杂度一定是 O(n) | 表尾插入/删除是 O(1) | 只有表头或中间位置才是 O(n) |
| 链表插入删除时间复杂度是 O(1) | 查找位置 O(n)，修改指针 O(1) | 总时间复杂度为 O(n) |
| 线性表每个元素都有前驱和后继 | 表头无前驱，表尾无后继 | 这是线性表的基本特征 |
| 循环链表不是线性表 | 循环链表是线性表 | 循环只是存储方式，逻辑结构仍是线性 |
| 静态链表不支持插入删除 | 静态链表支持插入删除 | 只需修改 next 指针，无需移动元素 |

## 思维拓展

**问题 1**：若线性表的长度基本稳定，但需要频繁地按序号访问元素，应选择哪种存储结构？为什么？

**思考方向**：
- 顺序表支持随机访问，时间复杂度 O(1)
- 链表只能顺序访问，时间复杂度 O(n)
- 长度稳定意味着不需要频繁扩容

**答案**：应选择顺序表。因为顺序表支持随机访问，且长度稳定不需要频繁扩容，正好发挥顺序表的优势。

---

**问题 2**：若线性表的长度变化较大，且需要频繁地在任意位置插入和删除元素，应选择哪种存储结构？为什么？

**思考方向**：
- 链表插入删除只需修改指针，时间复杂度 O(1)（不计查找位置）
- 顺序表插入删除需要移动元素，时间复杂度 O(n)
- 链表动态分配内存，长度变化灵活

**答案**：应选择链表。因为链表插入删除效率高，且动态分配内存，适合长度变化大的场景。

---

**问题 3**：设计一个算法，判断单链表是否有环。要求时间复杂度 O(n)，空间复杂度 O(1)。

**思考方向**：
- 使用快慢指针法
- 快指针每次走两步，慢指针每次走一步
- 若有环，快指针最终会追上慢指针

**答案**：
```c
bool HasCycle(LinkList L) {
    LNode *slow = L->next;
    LNode *fast = L->next;
    
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast)  // 快慢指针相遇，说明有环
            return true;
    }
    return false;  // 快指针到达表尾，说明无环
}
```

## 思维拓展

**问题**：若有 10000 个学生信息需要存储，要求支持以下操作：
1. 频繁地根据学号查询学生信息
2. 频繁地插入和删除学生信息
3. 按学号有序存储

请分析：应该选择顺序表还是链表？为什么？

**思考方向**：

1. **学号查询**：若主要操作是按学号随机查询，顺序表更优（O(1) 随机访问），链表需要 O(n) 遍历
2. **插入删除**：若频繁插入删除，链表更优（只需 O(1) 修改指针，但需先找到位置）
3. **有序存储**：若需按学号有序，顺序存储查找可用二分法 O(logn)，但插入删除需移动元素
4. **综合考虑**：对于第 3 种需求（按学号有序存储），可以采用"折中方案"——使用静态链表或结合两者特点的数据结构

**考研真题关联**（2019 年选择题第 2 题）：
- 考核点：链表的头插法和尾插法建表
- 关键点：头插法逆序建表，尾插法顺序建表
