## 3.1 栈

### 3.1.1 栈的基本概念（高频考点）

**栈（Stack）** 是一种特殊的线性表，其插入和删除操作只能在表尾进行。允许插入和删除的一端称为**栈顶（Top）**，另一端称为**栈底（Bottom）**。

**栈的特点**：后进先出（Last In First Out，LIFO）

**栈的基本操作**：

| 操作 | 说明 |
|------|------|
| `InitStack(&S)` | 初始化空栈 |
| `DestroyStack(&S)` | 销毁栈，释放空间 |
| `Push(&S, x)` | 入栈，将元素 x 压入栈顶 |
| `Pop(&S, &x)` | 出栈，将栈顶元素弹出并用 x 返回 |
| `GetTop(S, &x)` | 读取栈顶元素（不弹出） |
| `StackEmpty(S)` | 判断栈是否为空 |
| `StackLength(S)` | 求栈的长度 |

**栈的数学性质**：n 个不同元素进栈，出栈元素的不同排列的个数为 Catalan 数：$\frac{1}{n+1}C_{2n}^{n}$

### 3.1.2 栈的顺序存储结构（必考考点，重点掌握）

顺序栈使用一组连续的存储单元存放栈中的元素，并设置一个栈顶指针。

```c
#define MAXSIZE 100  // 定义栈的最大容量

typedef struct {
    ElemType data[MAXSIZE];  // 存放栈中元素
    int top;                  // 栈顶指针（指向栈顶元素）
} SqStack;
```

**栈顶指针初始化**：通常将 top 初始化为 -1，表示空栈

**顺序栈的基本操作**：

#### 1. 初始化

```c
// 初始化顺序栈
void InitStack(SqStack &S) {
    S.top = -1;  // 栈顶指针置为 -1，表示空栈
}
```

#### 2. 判断栈空

```c
// 判断顺序栈是否为空
bool StackEmpty(SqStack S) {
    return S.top == -1;
}
```

#### 3. 入栈（Push）

```c
// 将元素 x 压入栈顶
bool Push(SqStack &S, ElemType x) {
    if (S.top == MAXSIZE - 1)  // 栈满
        return false;
    S.data[++S.top] = x;  // 栈顶指针先加 1，再入栈
    return true;
}
```

**时间复杂度**：O(1)

#### 4. 出栈（Pop）

```c
// 弹出栈顶元素，用 x 返回
bool Pop(SqStack &S, ElemType &x) {
    if (S.top == -1)  // 栈空
        return false;
    x = S.data[S.top--];  // 先出栈，栈顶指针再减 1
    return true;
}
```

**时间复杂度**：O(1)

#### 5. 读取栈顶元素

```c
// 读取栈顶元素（不弹出）
bool GetTop(SqStack S, ElemType &x) {
    if (S.top == -1)  // 栈空
        return false;
    x = S.data[S.top];  // 仅读取栈顶元素
    return true;
}
```

**时间复杂度**：O(1)

#### 6. 求栈长

```c
// 返回顺序栈的元素个数
int StackLength(SqStack S) {
    return S.top + 1;
}
```

**栈满条件**：`S.top == MAXSIZE - 1`
**栈空条件**：`S.top == -1`
**栈长**：`S.top + 1`

### 3.1.3 栈的链式存储结构

链栈使用链表实现，栈顶在链表头部，便于入栈和出栈操作。

```c
typedef struct StackNode {
    ElemType data;              // 数据域
    struct StackNode *next;      // 指针域
} StackNode, *LinkStack;
```

**链栈的特点**：
- 不存在栈满的情况（内存足够即可）
- 空栈表现为头结点的 next 指针为 NULL
- 插入和删除操作都在栈顶进行

**链栈的基本操作**：

#### 1. 初始化

```c
// 初始化链栈（创建头结点）
void InitStack(LinkStack &S) {
    S = (StackNode *)malloc(sizeof(StackNode));
    if (S == NULL)  // 内存分配失败
        exit(1);
    S->next = NULL;  // 头结点指针域置空
}
```

#### 2. 判断栈空

```c
bool StackEmpty(LinkStack S) {
    return S->next == NULL;
}
```

#### 3. 入栈

```c
// 将元素 x 压入链栈
void Push(LinkStack &S, ElemType x) {
    StackNode *p = (StackNode *)malloc(sizeof(StackNode));
    p->data = x;
    p->next = S->next;  // 新结点插入栈顶
    S->next = p;
}
```

**时间复杂度**：O(1)

#### 4. 出栈

```c
// 弹出链栈顶元素，用 x 返回
bool Pop(LinkStack &S, ElemType &x) {
    if (S->next == NULL)  // 栈空
        return false;
    StackNode *p = S->next;  // p 指向栈顶结点
    x = p->data;
    S->next = p->next;  // 修改栈顶指针
    free(p);  // 释放结点空间
    return true;
}
```

**时间复杂度**：O(1)

### 3.1.4 栈的应用（重要）

#### 1. 表达式求值

**中缀表达式**：运算符在操作数中间，如 `a + b * c`

**后缀表达式（逆波兰式）**：运算符在操作数后面，如 `a b c * +`

**前缀表达式（波兰式）**：运算符在操作数前面，如 `+ a * b c`

**中缀转后缀算法（使用栈）**：

1. 从左到右扫描中缀表达式
2. 遇到操作数时输出
3. 遇到左括号时入栈
4. 遇到右括号时弹出栈中元素直到遇到左括号
5. 遇到运算符时：
   - 若优先级大于栈顶运算符，入栈
   - 若优先级小于等于栈顶运算符，弹出栈顶运算符直到栈顶运算符优先级低

**后缀表达式求值算法（使用栈）**：

1. 从左到右扫描后缀表达式
2. 遇到操作数时入栈
3. 遇到运算符时弹出两个操作数进行运算，结果入栈
4. 最终栈顶即为结果

#### 2. 递归与栈

**递归**：函数调用自身的过程

**递归调用栈**：每次递归调用都会将当前函数的运行环境（返回地址、局部变量等）压入栈中，递归返回时再弹出。

**递归的效率问题**：
- 递归调用需要额外的栈空间
- 递归层次过深可能导致栈溢出
- 递归算法的时间复杂度通常较高

**递归算法改写为非递归算法**：
- 使用栈模拟递归调用过程
- 保存递归调用的状态

#### 3. 括号匹配

**算法思想**：
1. 初始化一个空栈
2. 从左到右扫描表达式
3. 遇到左括号入栈
4. 遇到右括号时弹出栈顶元素进行匹配
5. 若匹配失败或栈空，则表达式无效
6. 扫描结束后，若栈为空，则表达式有效

```c
bool BracketMatch(char *exp) {
    SqStack S;
    InitStack(S);
    int i = 0;
    char c, top;
    while ((c = exp[i]) != '\0') {
        if (c == '(' || c == '[' || c == '{')
            Push(S, c);  // 左括号入栈
        else if (c == ')' || c == ']' || c == '}') {
            if (StackEmpty(S))  // 栈空，匹配失败
                return false;
            Pop(S, top);  // 弹出栈顶元素
            if ((c == ')' && top != '(') ||
                (c == ']' && top != '[') ||
                (c == '}' && top != '{'))  // 不匹配
                return false;
        }
        i++;
    }
    return StackEmpty(S);  // 栈空则匹配成功
}
```

### 3.1.5 本节试题精选

**选择题**

1. 一个栈的输入序列为 1, 2, 3, 4，则下列序列中不可能是栈的输出序列的是（ ）
   - A. 1, 4, 3, 2
   - B. 2, 3, 4, 1
   - C. 1, 3, 2, 4
   - D. 4, 3, 2, 1

2. 循环队列的优点是（ ）
   - A. 入队和出队操作时间复杂度低
   - B. 不需要移动数据
   - C. 不需要判断队满
   - D. 不需要判断队空

3. 一个栈的输入序列为 1, 2, 3, ..., n，输出序列的第一个元素是 n，则第 i 个输出元素是（ ）
   - A. n - i + 1
   - B. n - i - 1
   - C. i
   - D. n - i

4. 下列关于栈的叙述正确的是（ ）
   - A. 栈是一种先进先出的数据结构
   - B. 栈不是线性表
   - C. 栈可以顺序存储，也可以链式存储
   - D. 栈的插入操作在栈顶进行

**答案与解析**

1. **答案：C**
   - 解析：栈的输入序列为 1, 2, 3, 4，输出序列 1, 3, 2, 4 不可能是因为：当输出第一个元素为 1 时，2, 3, 4 都在栈中，栈顶为 4，不可能先输出 3 再输出 2（因为 3 在 2 下面，2 先出栈）。

2. **答案：B**
   - 解析：循环队列的优点是在队首和队尾进行插入和删除操作时，不需要移动数据元素，只需修改指针。

3. **答案：A**
   - 解析：输出序列第一个是 n，说明所有元素入栈后才输出。栈的输入序列是 1, 2, 3, ..., n，输出第一个是 n，则输出序列为 n, n-1, ..., 1，即第 i 个输出元素是 n-i+1。

4. **答案：D**
   - 解析：栈是一种先进后出的数据结构；栈是线性表；栈可以顺序存储也可以链式存储；栈的插入（入栈）和删除（出栈）操作都在栈顶进行。

## 3.2 队列

### 3.2.1 队列的基本概念（高频考点）

**队列（Queue）** 是一种特殊的线性表，其插入操作在表尾进行，删除操作在表头进行。允许插入的一端称为**队尾（Rear）**，允许删除的一端称为**队头（Front）**。

**队列的特点**：先进先出（First In First Out，FIFO）

**队列的基本操作**：

| 操作 | 说明 |
|------|------|
| `InitQueue(&Q)` | 初始化空队列 |
| `DestroyQueue(&Q)` | 销毁队列 |
| `EnQueue(&Q, x)` | 入队，将元素 x 加入队尾 |
| `DeQueue(&Q, &x)` | 出队，删除队头元素并用 x 返回 |
| `GetHead(Q, &x)` | 读取队头元素（不删除） |
| `QueueEmpty(Q)` | 判断队列是否为空 |
| `QueueLength(Q)` | 求队列的长度 |

### 3.2.2 队列的顺序存储结构（循环队列）（必考考点，重点掌握）

**普通顺序队列的问题**：随着入队和出队操作，队头和队尾指针不断后移，会导致"假溢出"问题——明明队列未满，却无法再入队。

**循环队列**：将顺序队列视为一个环，使得存储空间能够循环使用，解决"假溢出"问题。

**循环队列的表示**：

```c
#define MAXSIZE 100

typedef struct {
    ElemType data[MAXSIZE];  // 存放队列元素
    int front;  // 队头指针
    int rear;   // 队尾指针
} SqQueue;
```

**循环队列的关键公式**：
- 初始状态（空队列）：`front == rear`
- 队空条件：`front == rear`
- 队满条件：`(rear + 1) % MAXSIZE == front`
- 队列长度：` (rear - front + MAXSIZE) % MAXSIZE`
- 入队：`rear = (rear + 1) % MAXSIZE`
- 出队：`front = (front + 1) % MAXSIZE`

**为什么牺牲一个存储单元？**
为了区分队空和队满，循环队列通常会浪费一个存储单元。当 rear 的下一个位置是 front 时，认为队满。

**循环队列的基本操作**：

#### 1. 初始化

```c
// 初始化循环队列
void InitQueue(SqQueue &Q) {
    Q.front = Q.rear = 0;  // 队头队尾指针指向 0
}
```

#### 2. 判断队空

```c
// 判断循环队列是否为空
bool QueueEmpty(SqQueue Q) {
    return Q.front == Q.rear;
}
```

#### 3. 判断队满

```c
// 判断循环队列是否为满
bool QueueFull(SqQueue Q) {
    return (Q.rear + 1) % MAXSIZE == Q.front;
}
```

#### 4. 入队

```c
// 将元素 x 加入队尾
bool EnQueue(SqQueue &Q, ElemType x) {
    if (QueueFull(Q))  // 队满
        return false;
    Q.data[Q.rear] = x;  // 元素入队
    Q.rear = (Q.rear + 1) % MAXSIZE;  // 队尾指针后移
    return true;
}
```

**时间复杂度**：O(1)

#### 5. 出队

```c
// 删除队头元素，用 x 返回
bool DeQueue(SqQueue &Q, ElemType &x) {
    if (QueueEmpty(Q))  // 队空
        return false;
    x = Q.data[Q.front];  // 保存队头元素
    Q.front = (Q.front + 1) % MAXSIZE;  // 队头指针后移
    return true;
}
```

**时间复杂度**：O(1)

#### 6. 读取队头元素

```c
// 读取队头元素（不删除）
bool GetHead(SqQueue Q, ElemType &x) {
    if (QueueEmpty(Q))  // 队空
        return false;
    x = Q.data[Q.front];
    return true;
}
```

#### 7. 求队列长度

```c
// 返回循环队列的元素个数
int QueueLength(SqQueue Q) {
    return (Q.rear - Q.front + MAXSIZE) % MAXSIZE;
}
```

### 3.2.3 队列的链式存储结构

链队列使用链表实现，队头在链表头部，队尾在链表尾部。

```c
typedef struct QNode {
    ElemType data;          // 数据域
    struct QNode *next;     // 指针域
} QNode, *QueuePtr;

typedef struct {
    QueuePtr front;  // 队头指针（指向头结点）
    QueuePtr rear;   // 队尾指针
} LinkQueue;
```

**链队列的特点**：
- 不存在队满的情况
- 空队列表现为 front 和 rear 都指向头结点
- 入队在队尾进行，出队在队头进行

**链队列的基本操作**：

#### 1. 初始化

```c
// 初始化链队列（创建头结点）
void InitQueue(LinkQueue &Q) {
    Q.front = Q.rear = (QNode *)malloc(sizeof(QNode));  // 创建头结点
    if (Q.front == NULL)  // 内存分配失败
        exit(1);
    Q.front->next = NULL;  // 头结点指针域置空
}
```

#### 2. 判断队空

```c
bool QueueEmpty(LinkQueue Q) {
    return Q.front == Q.rear;
}
```

#### 3. 入队

```c
// 将元素 x 加入队尾
void EnQueue(LinkQueue &Q, ElemType x) {
    QNode *p = (QNode *)malloc(sizeof(QNode));  // 创建新结点
    p->data = x;
    p->next = NULL;
    Q.rear->next = p;  // 将新结点插入队尾
    Q.rear = p;        // 修改队尾指针
}
```

**时间复杂度**：O(1)

#### 4. 出队

```c
// 删除队头元素，用 x 返回
bool DeQueue(LinkQueue &Q, ElemType &x) {
    if (Q.front == Q.rear)  // 队空
        return false;
    QNode *p = Q.front->next;  // p 指向队头结点
    x = p->data;
    Q.front->next = p->next;  // 修改头结点的 next 指针
    if (Q.rear == p)  // 如果删除的是最后一个结点
        Q.rear = Q.front;  // 修改队尾指针指向头结点
    free(p);  // 释放结点空间
    return true;
}
```

**时间复杂度**：O(1)

#### 5. 销毁链队列

```c
// 销毁链队列
void DestroyQueue(LinkQueue &Q) {
    while (Q.front != NULL) {
        Q.rear = Q.front->next;
        free(Q.front);
        Q.front = Q.rear;
    }
}
```

### 3.2.4 队列的应用（重要）

#### 1. 树的层次遍历

使用队列实现二叉树的层次遍历：
1. 将根结点入队
2. 取出队头结点，访问该结点
3. 若该结点有左孩子，将左孩子入队
4. 若该结点有右孩子，将右孩子入队
5. 重复步骤 2-4 直到队列为空

```c
void LevelOrder(BiTree T) {
    LinkQueue Q;
    InitQueue(Q);
    if (T == NULL) return;
    EnQueue(Q, T);  // 根结点入队
    while (!QueueEmpty(Q)) {
        BiTree p;
        DeQueue(Q, p);  // 队头结点出队
        visit(p);  // 访问结点
        if (p->lchild) EnQueue(Q, p->lchild);  // 左孩子入队
        if (p->rchild) EnQueue(Q, p->rchild);  // 右孩子入队
    }
}
```

#### 2. 图的广度优先遍历（BFS）

使用队列实现图的广度优先遍历：
1. 将起始顶点入队
2. 取出队头顶点，访问该顶点
3. 将与该顶点相邻且未访问的顶点入队
4. 重复步骤 2-3 直到队列为空

#### 3. 打印杨辉三角

使用队列打印杨辉三角：
1. 将第一行的 1 入队
2. 取出队头元素并打印
3. 计算下一行的值：将队头元素取出，与下一个元素相加得到新元素
4. 将新元素入队
5. 重复步骤 2-4 直到达到指定行数

#### 4. 主机间的简单通信（先进先服务）

队列可以实现主机间的先进先出服务，如作业调度、打印任务队列等。

### 3.2.5 易错点辨析（考研高频陷阱）

- **易错1**：循环队列队空和队满的判断——初始状态 `front == rear` 表示队空；`(rear + 1) % MAXSIZE == front` 表示队满
- **易错2**：循环队列长度的计算——`(rear - front + MAXSIZE) % MAXSIZE`，考虑 rear < front 的情况
- **易错3**：链队列删除最后一个结点——需要修改 rear 指针指向 front
- **易错4**：栈和队列的操作位置混淆——栈的插入和删除都在栈顶，队列的插入在队尾、删除在队头
- **易错5**：循环队列牺牲一个存储单元——这是为了区分队空和队满，是经典设计模式

### 3.2.6 本节试题精选

**选择题**

1. 下列关于循环队列的说法正确的是（ ）
   - A. 循环队列不会产生假溢出
   - B. 循环队列不需要判断队空
   - C. 循环队列不需要判断队满
   - D. 循环队列的入队和出队操作都不需要移动元素

2. 设循环队列的存储空间为数组 data[0..MAXSIZE-1]，队头 front 指向队头元素，队尾 rear 指向队尾元素的下一个位置，则该队列的队满条件是（ ）
   - A. front == rear
   - B. (rear + 1) % MAXSIZE == front
   - C. rear == MAXSIZE - 1
   - D. front == 0

3. 假设循环队列不牺牲一个存储单元，则无法区分（ ）
   - A. 队空和队满
   - B. 队空和队中有一个元素
   - C. 队中有一个元素和队满
   - D. 以上都不对

4. 下列数据结构中，适合作为打印任务队列的是（ ）
   - A. 栈
   - B. 队列
   - C. 树
   - D. 图

5. 一个链队列的队头和队尾指针分别为 front 和 rear，则判断队空的条件是（ ）
   - A. front == NULL
   - B. rear == NULL
   - C. front == rear
   - D. front != rear

6. 在一个链队列中，假设删除一个元素的代价是 O(1)，则（ ）
   - A. 入队和出队都是 O(1)
   - B. 入队是 O(1)，出队是 O(n)
   - C. 入队是 O(n)，出队是 O(1)
   - D. 入队和出队都是 O(n)

**答案与解析**

1. **答案：D**
   - 解析：循环队列通过取模运算解决了假溢出问题；循环队列仍需要判断队空和队满；循环队列的入队和出队操作都是通过修改指针实现的，不需要移动元素。

2. **答案：B**
   - 解析：循环队列的队满条件是 rear 的下一个位置是 front，即 `(rear + 1) % MAXSIZE == front`。

3. **答案：A**
   - 解析：如果不牺牲一个存储单元，队空条件 `front == rear` 和队满条件 `front == rear` 相同，无法区分。

4. **答案：B**
   - 解析：打印任务通常按照提交顺序执行，符合队列的先进先出特点。

5. **答案：C**
   - 解析：链队列通常带有头结点，队空条件是 `front == rear`（头结点和尾结点指向相同）。

6. **答案：A**
   - 解析：链队列的入队操作在队尾进行，出队操作在队头进行，都是修改指针的操作，时间复杂度均为 O(1)。

## 3.3 数组和特殊矩阵

### 3.3.1 数组的定义

**数组** 是由 n（n≥1）个相同数据类型的数据元素组成的有限序列。数组的元素可以是线性表的基本元素，也可以是线性表（形成多维数组）。

**数组的特点**：
- 数组元素占用的空间相同
- 可以通过下标直接访问任意元素（随机访问）
- 数组一旦建立，结构确定，不适合频繁的插入删除操作

**一维数组**：线性表结构，数据元素可以是基本数据类型或结构体

**二维数组**：可以看作"元素为一维数组的一维数组"，或"元素为线性表的线性表"

### 3.3.2 数组的存储结构

#### 1. 一维数组的存储结构

一维数组的元素按顺序存储在连续的内存空间中。

**地址计算**：设每个元素占用的存储单元数为 L（元素大小），第一个元素的首地址为 LOC(a₀)，则第 i 个元素的首地址为：

$$LOC(a_i) = LOC(a_0) + i \times L$$

#### 2. 二维数组的存储结构

二维数组有两种存储方式：

**行优先存储（C、PASCAL 等语言使用）**：
- 先存储第一行的所有元素，再存储第二行的所有元素，依此类推
- 元素 a[i][j] 的存储地址：

$$LOC(a_{i,j}) = LOC(a_{0,0}) + (i \times n + j) \times L$$

其中 n 是每行的列数。

**列优先存储（Fortran 等语言使用）**：
- 先存储第一列的所有元素，再存储第二列的所有元素，依此类推
- 元素 a[i][j] 的存储地址：

$$LOC(a_{i,j}) = LOC(a_{0,0}) + (j \times m + i) \times L$$

其中 m 是每列的行数。

**注意**：408考试中默认使用行优先存储。

### 3.3.3 特殊矩阵的压缩存储（重要）

**压缩存储**：为多个值相同的元素只分配一个存储空间，对零元素不分配空间。

#### 1. 对称矩阵

**对称矩阵**：矩阵中所有元素满足 a[i][j] = a[j][i]（1 ≤ i, j ≤ n）

**压缩存储方法**：只存储上三角区（或下三角区）的元素，包括主对角线。

**存储元素个数**：

$$1 + 2 + ... + n = \frac{n(n+1)}{2}$$

**地址计算**（行优先，存储下三角区）：

若 a[i][j]（i ≥ j）在下三角区：

$$LOC(a_{i,j}) = LOC(a_{1,1}) + \frac{i(i-1)}{2} + (j-1)) \times L$$

若 a[i][j]（i < j）在上三角区，则访问 a[j][i]。

#### 2. 三角矩阵

**上三角矩阵**：主对角线以下的元素均为常数 c（通常为 0）

**下三角矩阵**：主对角线以上的元素均为常数 c（通常为 0）

**压缩存储方法**：
- 存储 n(n+1)/2 个三角区元素
- 再存储 1 个常数 c

**地址计算**（以行优先存储上三角矩阵为例）：

若 a[i][j]（i ≤ j）在上三角区：

$$LOC(a_{i,j}) = LOC(a_{1,1}) + \frac{(i-1)(2n-i+2)}{2} + (j-i)) \times L$$

若 a[i][j]（i > j）在下三角区，则访问常数 c。

#### 3. 稀疏矩阵

**稀疏矩阵**：矩阵中非零元素的个数远远小于矩阵元素的总数（通常认为非零元素个数小于总元素个数的 5%）。

**压缩存储方法**：

**三元组表示法**：存储 (行号, 列号, 值) 的三元组。

```c
#define MAXSIZE 1000  // 非零元素的最大个数

typedef struct {
    int row;    // 行下标
    int col;    // 列下标
    ElemType e; // 元素值
} Triple;

typedef struct {
    Triple data[MAXSIZE + 1];  // 三元组表（下标从 1 开始）
    int mu, nu, tu;            // 矩阵的行数、列数、非零元素个数
} TSMatrix;
```

**十字链表表示法**：每个非零元素结点同时存储行链表指针和列链表指针，便于行列操作。

```c
typedef struct OLNode {
    int row, col;              // 行下标和列下标
    ElemType value;            // 元素值
    struct OLNode *right, *down;  // 指向同行下一非零元素和同列下一非零元素
} OLNode, *OLink;
```

**稀疏矩阵的缺点**：不能根据下标直接访问任意元素，访问效率低。

### 3.3.4 易错点辨析（考研高频陷阱）

- **易错1**：二维数组地址计算——C语言使用行优先存储，Fortran使用列优先存储，408默认行优先
- **易错2**：对称矩阵压缩存储——只存储下三角（或上三角）元素，访问上三角元素时需要转换为下三角
- **易错3**：三角矩阵压缩存储——除了存储三角区元素，还需要存储一个常数
- **易错4**：稀疏矩阵存储代价——三元组存储不能随机访问，访问效率低
- **易错5**：数组下标从 0 还是从 1 开始——地址计算时需要注意，C语言数组下标从0开始，但408考试中矩阵下标通常从1开始

### 3.3.5 本节试题精选

**选择题**

1. 设二维数组 A[0..m-1][0..n-1]（m 行 n 列）按行优先存储在连续内存中，每个元素占用 L 个存储单元，则元素 A[i][j] 的存储地址是（ ）
   - A. LOC(A[0][0]) + i*n + j) * L
   - B. LOC(A[0][0]) + (i*m + j) * L
   - C. LOC(A[0][0]) + (j*n + i) * L
   - D. LOC(A[0][0]) + (i + j) * L

2. 一个 n×n 的对称矩阵，采用压缩存储，需要存储的元素个数是（ ）
   - A. n²
   - B. n²/2
   - C. n(n+1)/2
   - D. n(n-1)/2

3. 一个上三角矩阵，采用压缩存储，若按行优先存储下三角部分，则第 i 行第 j 列（i < j）的元素存储在（ ）
   - A. 第 (i-1)(2n-i+2)/2 + j-i 个位置
   - B. 第 i(i+1)/2 + j 个位置
   - C. 第 j(j+1)/2 + i 个位置
   - D. 第 (i-1)(2n-i+2)/2 + j 个位置

4. 稀疏矩阵的三元组表示法中，存储的信息包括（ ）
   - A. 行数、列数、非零元素个数
   - B. 行数、列数、非零元素个数、非零元素值
   - C. 行下标、列下标、非零元素值
   - D. 以上都不对

5. 设有一个 10 阶的对称矩阵 A，采用压缩存储（按行优先存储下三角），则 A[8][5]（假设下标从1开始）存储在数组 S 中的位置是（ ）
   - A. 36
   - B. 37
   - C. 41
   - D. 42

6. 二维数组 A[3][4] 按行优先存储，每个元素占用 4 个字节，已知 A[0][0] 的地址为 100，则 A[2][3] 的地址是（ ）
   - A. 140
   - B. 144
   - C. 148
   - D. 152

**答案与解析**

1. **答案：A**
   - 解析：按行优先存储，元素 A[i][j] 前面有 i 行（每行 n 个元素），即 i×n 个元素，再加上第 i 行的前 j 个元素，因此地址为 LOC(A[0][0]) + (i×n + j) × L。

2. **答案：C**
   - 解析：对称矩阵只需存储下三角（或上三角）元素，包括主对角线。元素个数为 1+2+...+n = n(n+1)/2。

3. **答案：A**
   - 解析：上三角矩阵按行优先存储下三角部分，第 i 行第 j 列（i < j）的元素存储在第 (i-1)(2n-i+2)/2 + j-i 个位置。

4. **答案：A**
   - 解析：稀疏矩阵的三元组表示法需要存储行数(mu)、列数(nu)、非零元素个数(tu)，以及每个非零元素的行下标、列下标和值。

5. **答案：B**
   - 解析：按行优先存储下三角，A[8][5] 在第 8 行第 5 列（8 > 5），是下三角元素。存储位置 = 1+2+...+7 + 5 = 28 + 5 = 33 + 4 = 37（考虑数组下标从1开始）。

6. **答案：A**
   - 解析：按行优先存储，A[2][3] 前面有 2 行（每行 4 个元素），即 8 个元素，再加上第 2 行的前 3 个元素，即 11 个元素。地址 = 100 + 11 × 4 = 144。

## 归纳总结

本章介绍了栈、队列和数组三种数据结构，是数据结构课程的核心章节。

**核心考点**：

1. **栈**：
   - 后进先出（LIFO）特点
   - 顺序栈和链栈的实现
   - 栈的应用：表达式求值、递归、括号匹配
   - 入栈出栈操作时间复杂度 O(1)

2. **队列**：
   - 先进先出（FIFO）特点
   - 循环队列：解决假溢出问题
   - 循环队列队空队满判断：`front == rear`（空），`(rear+1)%MAXSIZE == front`（满）
   - 链队列的实现

3. **数组**：
   - 数组的存储结构：行优先、列优先
   - 对称矩阵压缩存储：存储下三角 n(n+1)/2 个元素
   - 三角矩阵压缩存储：存储三角区元素加一个常数
   - 稀疏矩阵压缩存储：三元组表示法、十字链表表示法

**典型错误辨析**：

| 错误说法 | 正确说法 | 解析 |
|---------|---------|------|
| 循环队列不需要判断队满 | 循环队列需要判断队满 | 牺牲一个存储单元后，用 `front == rear` 判断队空，`(rear+1)%MAXSIZE == front` 判断队满 |
| 栈可以先进先出 | 栈是后进先出 | 栈的特点是 LIFO，队列才是 FIFO |
| 稀疏矩阵可以用下标随机访问 | 稀疏矩阵不能用下标随机访问 | 三元组存储只能顺序访问，效率低 |
| 对称矩阵压缩存储需要 n² 个元素 | 对称矩阵压缩存储需要 n(n+1)/2 个元素 | 只需存储下三角或上三角元素 |

## 思维拓展

**问题**：如何用两个栈实现一个队列？请分析其入队、出队操作的时间复杂度。

**思考方向**：

1. **栈实现队列的思路**：使用两个栈，一个用于入队（栈A），一个用于出队（栈B）
2. **入队操作**：直接 Push 到栈A
3. **出队操作**：如果栈B不为空，直接 Pop 栈B；如果栈B为空，将栈A的所有元素逐个 Pop 并 Push 到栈B，再 Pop 栈B
4. **时间复杂度分析**：
   - 入队：O(1)
   - 出队：均摊 O(1)（每次元素最多移动一次）
5. **空间效率**：两个栈的总空间与队列长度成正比

**考研真题关联**（2018 年选择题第 3 题）：
- 考核点：栈的出栈序列合法性
- 关键点：理解栈的 LIFO 特性，判断给定序列是否能由给定输入序列得到
